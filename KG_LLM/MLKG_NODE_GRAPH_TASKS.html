LOCAL AND EGONET FEATURES MODULE_V1.0
Local Features

Goal: Counting the node's direct connections.

  • Individual Focus: Focus on individual nodes and their immediate adjacency.
  • Metric Role: These metrics describe a node's specific role or importance based on its direct connections without considering the broader neighborhood structure.
  • Example: Degree Centrality (count of edges).
  • Focus: The specific node and its incident edges.
  • Key Question: "How many direct connections does this point have?"
Egonet Features

Goal: Focus on the subgraph induced by a node and its neighbors (the "ego" and its "alters").

  • Structural Patterns: These capture local clustering, density, and the complex structural patterns within a node's social circle.
  • Relationships: Egonet Features look at the relationships between those connections.
  • Neighborhood: The structure and "tightness" of the neighborhood.
  • Key Question: "Are my friends also friends with each other?"
NODE-FOCUSED TASKS MODULE_V1.1
Input & Output

Input: A single graph where individual nodes and their relationships (edges) serve as the data points.

Output: Predictions for specific components, such as assigning classes to nodes (classification) or determining the probability of a connection between them (link prediction).

Node Classification
  • Homophily: The tendency for connected nodes to share similar attributes (e.g., social circles).
  • Structural Equivalence: Nodes with similar neighborhood patterns often share roles, like proteins with similar functions.
  • Heterophily: A pattern where "opposites attract," and nodes connect to those with different traits.
  • Multilabel Classification: Assigning multiple categories to one node, such as predicting several interest groups for a single Flickr user.
Link Prediction

Link prediction uses existing network structure and node traits to forecast missing or future connections. It treats the graph as a semi-supervised problem to fill in data gaps.

  • Recommendations: Suggesting friends or products.
  • Knowledge Graph Completion: Inferring new facts or relationships.
  • Bioinformatics: Predicting drug side effects or protein interactions.
Community detection (or clustering)

Identifies dense groups of nodes that are more connected to each other than to the rest of the network.

  • Structural Summarization: It provides a macro-view of massive graphs by grouping them into functional modules or institutions.
  • Topological Basis: Groups are formed because nodes in the same "community" have a higher propensity to interact (e.g., researchers at the same university).
  • Hybrid Approaches: While mostly unsupervised, methods like label propagation can use a few known labels to guide the discovery of these communities.
GRAPH-FOCUSED TASKS MODULE_V1.2
Global Prediction

Input: A collection of multiple independent graphs used as distinct data points.

Output: A single label or value assigned to an entire graph structure, such as classifying a whole molecule or predicting overall graph functions.

ML on Graphs: MODULE_V1.3
Representation Learning

Graph embedding or representation learning. By converting non-Euclidean graph structures (nodes/edges) into standard numerical vectors (feature engineering), you can feed them into traditional models like Random Forests or Neural Networks.

  • Vectorization: Vectorization in graphs is the process of converting complex relational data into numerical vectors.
  • Node/Edge Attributes: If a node has raw data (e.g., a user's age), that specific feature is encoded.
  • Topological Features: The relationships/structure itself (e.g., how many neighbors a node has) is "summarized" into the same vector.
  • Feature Engineering: Feature Engineering is critical: How nodes are represented is more critical than the choice of classifier.
  • Domain Knowledge: Understanding network dynamics (like homophily) often outperforms generic metrics like simple degree centrality.
Evaluating ML Graphs:

The process transforms test node embeddings using the same scaling parameters as the training set to ensure consistency. The evaluation utilizes four key statistical metrics:

  • Precision: The accuracy of positive predictions.
  • Recall: The ability to find all relevant instances.
  • F-score: The harmonic mean of precision and recall, balancing both.
  • Confusion Matrix: A table summarizing where the model correctly identifies classes versus where it misclassifies them.
Feature Engineering: MODULE_V1.4
Core Strategies

In traditional datasets, features are direct measurements from the real world. In graph-based ML, we must construct our features from the graph structure itself, we need to capture meaningful properties of nodes, relationships, or entire graphs as numerical values.

  • Local Features: These capture data from the ego-centered network (egonet). They analyze the "ego" (the specific node) and its "alters" (immediate or n-hop neighbors) to understand its local environment.
  • Global Features: These quantify a node's role within the entire network. This includes centrality metrics like PageRank, Betweenness, and Closeness, which measure a node's influence and its connectivity to the rest of the graph.
  • Incremental Featurization: Instead of giving the model all the data at once, they add one metric at a time. Building the feature vector step-by-step. You start with a simple vector (e.g., just node degree), train the model, and evaluate performance. Then, you concatenate a new metric (like PageRank) to that vector and re-train to see if accuracy improves.
Manual Node Feature Examples (Graph Theory)
  • Degrees: Node degree measures immediate reach and influence, making it a robust baseline for featurization.
  • Local homophily: By decomposing the degree into specific categories (e.g., connections to different groups), you capture the tendency for similar nodes to cluster.
  • Triangle Subgraphs: Triangle subgraphs quantify the clustering and density of a node's immediate neighborhood. Triangles reveal if those neighbors are also connected to each other.
    • Structural Strength: High triangle counts suggest the node is part of a cohesive group, which often correlates with higher influence or trust.
    • Homophily Amplification: If a node is part of multiple triangles where other members share a specific trait.
  • Density: It quantifies the intensity of interaction within a node's immediate neighborhood. A high density in an egonet suggests a highly cohesive group where information or influence spreads rapidly.
  • Geodesic Path: Shortest path represents the minimum distance between two nodes.
  • Closeness: Quantifies how efficiently a node can reach every other node in the network. Calculated as the reciprocal of the average shortest path distance.
    • Standard Closeness: High closeness score indicates a node that is "central".
    • Normalization: Handles disconnected graphs where distances might be infinite, adjusting scores to reflect reachable nodes.
  • Betweenness: Measures how often a node acts as a bridge between other nodes by appearing on the shortest paths between other pairs.
Manual Relationship Feature Examples:

Node-Based Combination: Compare the node embeddings. This approach creates a "link" feature by concatenating the data already known about the two individual nodes.

Symmetric and Distance-Based Operation:

  • Average: Captures the "central tendency" by taking the element-wise mean. Useful for community membership or social circle prediction.
  • L1 (Manhattan): Measures absolute difference to highlight how dissimilar nodes are across dimensions.
  • L2 (Euclidean): Calculates squared difference, emphasizing larger gaps between feature values.

Compositional Operations:

  • Concatenation: Joins vectors end-to-end. Preserves all data but doubles computational cost.

    • Gold standard when relationship order matters (e.g., "A acts on B").

  • Hadamard (Element-wise Product): Multiplies corresponding elements. Captures interactions where both nodes must be high to signal a relationship.
Path-Based Features:
  1. Semantic Blueprinting (Metapath Definition): Designing path templates through the graph schema. You map specific sequences, like Villain to Philosopher, to capture the underlying logic of the network.
  2. Metapath Reduction (Filtering the Columns): Uses statistical selection to filter these templates. You analyze which paths correlate with your target and discard the rest to keep the feature vector small and efficient.
  3. Structural Scoring (Calculating the Values): Calculates weights for the selected paths using metrics like DWPC. This prevents high-degree hubs from dominating the score, ensuring niche connections are valued properly.
Semi-Automated Feature Extraction: MODULE_V1.5
ReFeX - Recursive feature extraction

Goal: Recursively generating local and egonet features from the graph structure.

  • Aggregation: Aggregates egonet features into a summary statistic (mean or sum) and appends to the original vector.
  • Evolution: Through recursion, simple counts evolve into regional "signatures." Captures increasingly global structural patterns.
  • Structural Equivalence: Allows a node to learn its structural role and identifies "look-alike" nodes even across different graphs.
Key Pruning Techniques

To prevent combinatorial explosion:

  • Correlation Analysis: Discards features that move in lockstep to keep the model lean.
  • Logarithmic Binning: Groups values into "buckets" to handle "Power Law" distributions (many nodes with few connections vs few with millions).
  • Threshold-based Pruning: Removes features with almost no variation across the graph that lack the power to distinguish roles.
