DOMAIN SPECIFIC ANALYSIS

Goal: Transforming structured and semistructured schemas and data formats into a homogeneous graph. KGs transform scattered data into a unified map.

Conductivity (Conductance)

Measures how "leak-proof" a community is. It compares the number of edges leaving the group to the total edges inside it.

  • Low Value: A distinct, tight-knit clique (like a family) that barely interacts with outsiders.
  • High Value: A vague group that blends into the background noise of the network.
  • When to use: Use to validate cluster quality. If your fraud detection model groups nodes with high conductance, it likely failed to find a real criminal ring.

Intensity (Density)

Measures internal cohesiveness. It calculates the ratio of actual connections to potential connections within a group.

  • High Value: Everyone knows everyone. The group is highly collaborative and robust.
  • Low Value: A loose collection of acquaintances. The group is fragile and barely connected.
  • When to use: Use to assess group strength. A high-density subgraph in a transaction network often indicates a coordinated attack or a high-velocity money laundering cycle.

Reachability

Measures network integrity. It asks: "If I start at Node A, can I eventually reach Node Z?"

  • High Value: A "Giant Component" exists. Information flows freely across the entire system.
  • Low Value: A shattered system of isolated islands. Data is trapped in silos.
  • When to use: Use to analyze information flow. If a cyber-attack hits a network with low reachability, the damage is naturally contained; in high reachability, it spreads everywhere.
PATHS AND METAPATHS

Path

A specific sequence of connected nodes and edges leading from a start node (A) to a target node (B).

  • How it works: It traces a single, concrete route through the network topology (e.g., User A → Clicked → Item B). It represents one specific instance of connectivity or flow between two entities.
  • When to use / Why: Use when tracing specific forensic trails, transaction flows, or user journeys. It isolates exact causal links, allowing you to audit how two specific nodes are connected.

Metapath

An established schematic pattern of node and edge types that defines a specific semantic relationship (e.g., Author → Paper → Author).

  • How it works: It acts as a template or filter. Instead of looking at random connections, it enforces a semantic structure (like "Co-Authorship") by only considering paths that follow the specific sequence of types defined in the schema.
  • When to use / Why: Use in heterogeneous graphs to define "similarity" or meaningful relationships. It allows you to relate two nodes based on shared context (e.g., same industry) rather than just direct adjacency.

Path Count

The raw integer count of how many valid paths exist between two nodes that match a specific metapath pattern.

  • How it works: It iterates through the graph and sums up every distinct path that satisfies the metapath criteria. If User A and User B purchased 5 of the same items, the Path Count for the "User-Item-User" metapath is 5.
  • When to use / Why: Use for baseline feature engineering. It provides a raw magnitude of connection strength but acts naively, failing to account for "popularity bias" (connecting via a common hub vs. a niche node).
THE MATHEMATICS: PDP AND DWPC

The Path-Degree Product (PDP)

A metric for a single path that down-weights connections passing through high-degree "super-nodes."

$$PDP(path) = \prod_{d \in D_{path}} d^{-w}$$
  • How it works: It examines every node along a specific path and raises its degree ($d$) to a negative exponent ($-w$). Mathematically, this means paths through massive hubs (high $d$) yield tiny scores, while paths through exclusive, low-degree nodes yield high scores.
  • When to use / Why: Use to quantify the specificity or information content of a single connection. It ensures that widely connected nodes (like the word "the" or a generic item) don't artificially inflate similarity scores.

The Degree-Weighted Path Count (DWPC)

The sum of PDP scores for all paths matching a metapath. It represents the total "quality" of the relationship.

$$DWPC_m(s, t) = \sum_{path \in Paths_m(s, t)} PDP(path)$$
  • How it works: It aggregates the "exclusivity" of the relationship. It finds every unique path matching the pattern, calculates the PDP for each, and sums them. This rewards pairs connected by many specific paths while discounting those connected only by generic hubs.
  • When to use / Why: Use as a robust metric for semantic similarity (e.g., in drug discovery). It distinguishes between meaningful connections and noise, providing a weighted score that accurately reflects the strength of the association.
