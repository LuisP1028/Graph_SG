GRAPH PROPERTIES MODULE_V1.0
Directed graphs

Goal: Also called digraphs, each edge has a direction or orientation.

  • Key 01: This means that the edge connects two nodes in a particular direction, where one node is the source and the other is the destination.
Undirected graph

Goal: Where the edges have no direction.

  • Key 01: This means that the edge between two vertices can be traversed in either direction, and the order in which we visit the nodes does not matter.
Weighted Graphs

Goal: In a weighted graph, each edge has a weight or cost associated with it.

  • Key 01: These weights can represent various factors, such as distance, travel time, or cost.
  • Key 02: For example, in a transportation network, the weights of edges might represent the distances between different cities or the time it takes to travel between them.
  • Key 03: In contrast, unweighted graphs have no weight associated with their edges.
  • Key 04: These types of graphs are commonly used in situations where the relationships between nodes are binary, and the edges simply indicate the presence or absence of a connection between them.
Connected and Disconnected Graphs

Goal: Define the reachability between vertices within the graph topology.

  • Connected: In connected graphs, there is a path between any two vertices in the graph.
  • Disconnected: In disconnected graphs, at least two vertices are not connected by a path.
GRAPH TYPES MODULE_V1.0
Tree

Goal: A connected, undirected graph with no cycles.

  • Key 01: Since there is only one path between any two nodes in a tree, a tree is a special case of a graph.
  • Key 02: Trees are often used to model hierarchical structures, such as family trees, organizational structures, or classification trees.
Rooted Tree

Goal: A tree in which one node is designated as the root, and all other vertices are connected to it by a unique path.

  • Key 01: Rooted trees are often used in computer science to represent hierarchical data structures.
Directed acyclic graph (DAG)

Goal: A directed graph that has no cycles.

  • Key 01: This means that the edges can only be traversed in a particular direction, and there are no loops or cycles.
  • Key 02: DAGs are often used to model dependencies between tasks or events – for example in computing the critical path of a job.
Bipartite graph

Goal: Graph in which the vertices can be divided into two types, such that all edges connect vertices in different sets.

  • Key 01: Bipartite graphs are often used to model relationships between two different types of objects, such as buyers and sellers, or employees and jects.
Complete Graph

Goal: Graph in which every pair of vertices is connected by an edge.

  • Key 01: Complete graphs are often used in combinatorics to model problems involving all possible pairwise connections.
GRAPH MEASURES MODULE_V1.0
Degree centrality

Goal: One of the simplest and most commonly used measures of centrality.

  • Key 01: It is simply defined as the degree of the node.
  • Key 02: A high degree centrality indicates that a vertex is highly connected to other vertices in the graph, and thus significantly influences the network.
Closeness centrality

Goal: Measures how close a node is to all other nodes in the graph.

  • Key 01: It corresponds to the average length of the shortest path between the target node and all other nodes in the graph.
  • Key 02: A node with high closeness centrality can quickly reach all other vertices in the network.
Betweenness centrality

Goal: Measures the number of times a node lies on the shortest path between pairs of other nodes in the graph.

  • Key 01: A node with high betweenness centrality acts as a bottleneck or bridge between different parts of the graph.
Density

Goal: Density measures how saturated a network is with connections compared to its theoretical maximum.

  • Key 01: It represents the "compactness" of the topology: a density of 1.0 is a complete graph where everyone is connected to everyone else; a density near 0 is a sparse graph.
  • Key 02: Generally, a graph is considered dense if its density is greater than 0.5 and sparse if its density is less than 0.1.
CORE METHODS TO REPRESENT GRAPHS MODULE_V1.0
ADJACENCY MATRIX

Goal: A matrix that represents the edges in a graph, where each cell indicates whether there is an edge between two nodes.

  • Key 01: The matrix is a square matrix of size N × N, where N is the number of nodes in the graph.
  • Key 02: A value of 1 in the cell (i, j) indicates that there is an edge between node i and node j.
  • Advantages: Checking if two nodes are connected is O(1) constant time. Enables efficient matrix operations for algorithms (e.g., shortest paths via matrix powers).
  • Drawbacks: Space complexity is O(N²), where N is the number of nodes – N grows quadratically. Adding/removing nodes is costly (requires resizing or shifting the matrix). Inefficient for large or sparse graphs due to high memory use. Poor for dynamically changing graphs.
SYMMETRY & ASYMMETRY

Goal: Distinguish between Undirected and Directed matrix properties.

  • SYMMETRY: Undirected graph connections are two-way streets. If Node A is connected to Node B, then Node B is automatically connected to Node A. In the adjacency matrix, this means the value at row A, column B must be identical to the value at row B, column A. This creates a mirror image across the diagonal, making the matrix mathematically symmetric. Symmetry guarantees real eigenvalues and orthogonal eigenvectors.
  • ASYMMETRY: Directed graph relationships are "one-way" arrows. Node A can point to Node B (a 1 in the matrix) without Node B pointing back (a 0 in the matrix). This breaks the mirror, resulting in an asymmetric matrix. Used for causal inference or knowledge graphs.
EDGE LIST

Goal: An edge list is a registry of active connections.

  • Key 01: It ignores the "voids" where no edge exists, making it memory-efficient for sparse data.
  • Key 02: Edge lists prioritize storage efficiency; matrices prioritize computational speed and mathematical manipulation.
ADJACENCY LISTS

Goal: Adjacency lists store a mapping of each node to a list of its immediate neighbors.

  • Advantages: Space complexity O(V + E), where V = nodes, E = edges – very efficient for sparse graphs. Adding nodes or edges is typically O(1) constant time. Easy and fast to iterate over a node’s neighbors (great for traversal algorithms like DFS/BFS).
  • Drawbacks: Checking if two nodes are connected is O(degree) – slower than O(1) in adjacency matrix (can be O(V) in worst case). Less cache-friendly and slower for dense graphs compared to matrix. Requires more complex implementation (e.g., hash maps or lists per node).
